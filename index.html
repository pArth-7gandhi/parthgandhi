Project: Event-Driven ETL Pipeline Modernization

Problem Statement

Existing system ran 15 Python ETL pipelines every 30 minutes (each taking ~5 mins), causing database bottlenecks, high CPU/memory usage, and difficult debugging due to flooded log files.

Solution: Event-Driven Architecture

  • Replaced scheduled batches with event-driven architecture using Redis Streams as message broker
  • Built ReactJS UI allowing users to trigger pipeline jobs on-demand instead of time-based execution
  • Implemented distributed pipeline system: Common pipeline picks job from Redis → routes to module-specific queue → executes ETL transformations → writes to SQL Server
  • Separated monolith into microservices with isolated log files per module for better observability
  • Containerized services and implemented GitLab CI/CD for automated deployments
